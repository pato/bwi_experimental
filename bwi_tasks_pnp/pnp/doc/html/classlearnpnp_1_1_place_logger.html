Detailed Description

Keeps track of the value of specified markings, and writes them on a file.

Given a set of Markings' ids, it records which ones are visited on a given episode and logs their value on a file. The set of Markings the logger will take into account is specified to the constructor. Before the first episode, the method initialize() must be called, in order to initialise the log file. Then, through the method afterUpdate(), this PlaceLogger keeps track of the markings actually visited. Finally, when saving() is called, it stores the values of the specified markings at that time, or a '?' (question mark) if that marking has not been visited yet.

The markings that must be logged are specified through their id. A marking id is given by the ids of each place with a number of token higher than zero, separated by an '-' (hyphen) and in lexicographical order. For instance, let the marking we want to log have a non-zero number of tokens in the places p1, p2, p14, p32. Its marking would be "p1-p14-p2-p32". Please note the lexicographical order.
